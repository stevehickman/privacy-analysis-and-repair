# Regulations by jurisdiction

Reference catalogue for the `privacy-by-design` skill. Load this when a finding needs a regulation outside the ten most-cited (which are in SKILL.md), or when the user has set scope to a specific region and you want the complete picture for that region.

Cite by name with the article or section number. Only cite regulations within the scope set at the start of the review.

## European Union / EEA

| Regulation | Key provisions to cite |
|---|---|
| **GDPR (Reg. 2016/679)** | Art. 5 (principles), 6 (lawful basis), 7 (consent), 9 (special categories), 13–14 (notice), 15 (access), 17 (erasure), 22 (automated decisions), 25 (privacy by design and by default), 28 (processors), 32 (security), 33–34 (breach notification), 35 (DPIA), 44+ (international transfers) |
| **ePrivacy Directive 2002/58** | Art. 5(3) — consent for non-essential cookies / trackers / device-storage access |
| **AI Act (Reg. 2024/1689)** | Prohibited practices (Art. 5), high-risk system obligations, general-purpose AI transparency duties |
| **Digital Services Act (Reg. 2022/2065)** | Dark-pattern prohibition, advertising transparency, targeting protections for minors |
| **Digital Markets Act (Reg. 2022/1925)** | Gatekeeper obligations on data combination and cross-service use |
| **NIS2 Directive** | Cybersecurity obligations for essential and important entities |
| **Data Act (Reg. 2023/2854)** | User access to and portability of data generated by connected products |

|---|---|

| **TCPA** | Phone calls, SMS, autodialler restrictions |
| **CAN-SPAM** | Commercial email |
| **FTC Act §5** | Unfair or deceptive practices — the FTC's general privacy enforcement hook |

## United States — state omnibus consumer privacy laws

| State | Statute | Notable provisions |
|---|---|---|
| California | **CCPA / CPRA** | Sensitive PI category; right to limit; opt-out of sale/share; automated-decision-making rules; CPPA enforcement; §1798.100 (access), .105 (deletion), .120 (opt-out), .121 (sensitive PI), .135 (notice) |
| Virginia | **VCDPA** | Opt-in for sensitive PI; controller/processor distinction |
| Colorado | **CPA** | Universal opt-out signal (Global Privacy Control) recognition |
| Connecticut | **CTDPA** | Closely aligned with CPA |
| Utah | **UCPA** | Narrower scope; opt-out model |
| Texas | **TDPSA** | Broad applicability via small-business carve-out |
| Oregon | **OCPA** | Right to know specific third-party recipients of one's data |
| Iowa | **ICDPA** | Narrower, business-friendly variant |
| Indiana | **INCDPA** | Closely aligned with VCDPA |
| Tennessee | **TIPA** | Safe harbour for written privacy programmes |
| Montana | **MCDPA** | Smaller-population threshold for applicability |
| Delaware | **DPDPA** | Universal opt-out signal recognition |
| New Hampshire | **NHDPA** | Closely aligned with CTDPA |
| New Jersey | **NJDPA** | Sensitive-data opt-in; financial info treated as sensitive |
| Minnesota | **MCDPA** | Profiling notice and explainability obligations |
| Rhode Island | **Data Transparency and Privacy Protection Act** | Disclosure-heavy model |
| Maryland | **MODPA** | Notably strict — data minimisation by statute, broad sensitive-data definition |
| Kentucky | **KCDPA** | Closely aligned with VCDPA |
| Florida | **Florida Digital Bill of Rights** | Narrow scope (very large businesses), but novel provisions on government-pressured moderation |

Effective dates and applicability thresholds shift constantly; check current status when a finding turns on a specific state's law.

## United States — state sector / specialty laws

| Statute | Where to cite |
|---|---|
| **BIPA** (Illinois) | Biometric Information Privacy Act — written consent before biometric collection; statutory damages drove the Facebook ($650M) and TikTok ($92M) settlements; the canonical biometric anchor nationwide |
| **My Health My Data Act** (Washington) | Broad consumer-health-data definition including inferences; private right of action |
| **California AADC** | Children's design code (subject to ongoing litigation) |
| **NY SHIELD Act** | Reasonable-security obligations; breach notification |
| **NYDFS Cybersecurity Regulation (23 NYCRR 500)** | Cybersecurity-programme requirements for financial-services firms |
| **California Delete Act (SB 362)** | Data-broker registration and one-stop deletion mechanism |
| **Connecticut data-broker law** | Registration; deletion request handling |
| **Illinois Genetic Information Privacy Act (GIPA)** | Restrictions on collection and disclosure of genetic information |
| **Massachusetts 201 CMR 17** | Comprehensive written information security programme requirement |

## Cross-cutting international frameworks

| Regime | Where to cite |
|---|---|
| **OECD Privacy Principles** | Pre-statutory baseline when no single regulation applies |
| **Convention 108+** | Council of Europe data-protection convention |
| **APEC CBPR (Cross-Border Privacy Rules)** | Cross-border privacy rules for Asia-Pacific transfers |
| **Global CBPR Forum** | Successor / extension to APEC CBPR |
| **EU–US Data Privacy Framework** | Adequacy mechanism for EU→US transfers (successor to Privacy Shield) |
| **EU Standard Contractual Clauses (SCCs)** | EU mechanism for transfers to non-adequate countries |
| **UK International Data Transfer Agreement (IDTA)** | UK transfer mechanism |
| **Binding Corporate Rules (BCRs)** | Intra-group transfer mechanism |
| **ISO/IEC 27701** | Privacy Information Management System extension to 27001 — useful as an assurance reference |
| **ISO/IEC 29100** | Privacy framework — terminology and principles |
| **NIST Privacy Framework** | US-anchored voluntary framework — useful in US federal-contractor contexts |
